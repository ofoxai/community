# English

## The Code

```
1. Ship > Talk
2. Code wins arguments
3. Done > Perfect
4. Users > Opinions
5. Simple > Clever
```

## The Rules

### 🚀 Ship It
- Working code beats perfect slides
- Iterate in public, not in private
- If it's not deployed, it doesn't exist

### 🔧 Build It Right
- Read the docs before asking
- Reproduce before reporting
- PR > Issue > DM

### 🤝 Respect the Craft
- Every bug report is a gift
- Every PR deserves a review
- Every question was once yours

### ⚡ Move Fast
- Bias toward action
- Small PRs, fast merges
- Revert > Debate

### 🧠 Think AI-Native
- LLMs are tools, not magic
- Prompt engineering is engineering
- Cost per token matters

# 中文

- 报告前先复现
- PR > Issue > 私信

- 每个 Bug 报告都是礼物

- 每个问题你也曾问过

- 偏向行动
- 小 PR，快合并
- 回滚 > 争论

### 🧠 AI 原生思维
- LLM 是工具，不是魔法
- Prompt 工程也是工程
- Token 成本很重要

# 日本語

## コード

```
1. 出荷 > 議論
2. コードが論争に勝つ
3. 完了 > 完璧
4. ユーザー > 意見
5. シンプル > クレバー
```

## ルール

### 🚀 出荷せよ
- 動くコードは完璧なスライドに勝る
- 公開でイテレーション、非公開ではなく
- デプロイされていなければ存在しない

### 🔧 正しく作れ
- 質問前にドキュメントを読む
- 報告前に再現する
- PR > Issue > DM

### 🤝 技術を尊重せよ
- すべてのバグ報告は贈り物
- すべての PR はレビューに値する
- すべての質問はかつて自分もした

### ⚡ 速く動け
- 行動を優先
- 小さな PR、素早いマージ
- リバート > 議論

### 🧠 AI ネイティブ思考
- LLM はツール、魔法ではない
- プロンプトエンジニアリングもエンジニアリング
- トークンコストは重要
